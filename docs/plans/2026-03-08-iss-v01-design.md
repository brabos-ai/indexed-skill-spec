# ISS v0.1 — Design Document

> Indexed Skills Specification — padrão global para leitura cirúrgica de documentação por agentes AI.

## Problema

Skills com milhares de linhas são carregadas inteiramente quando o agente precisa de apenas uma fração. Isso desperdiça contexto, aumenta latência e custo.

## Solução

Um sistema de indexação inline baseado em tiers que permite ao agente:
1. Detectar que a skill é indexada (via frontmatter)
2. Ler o índice (~poucas linhas) para decidir o que carregar
3. Ler cirurgicamente só as seções relevantes

## Detecção

Campo `indexed-skill` no frontmatter YAML:

```yaml
indexed-skill: tier 1   # arquivo único
indexed-skill: tier 2   # múltiplos arquivos
```

Sem esse campo → skill convencional, lê normalmente.

## Tier 1 — Arquivo único

Todo o conteúdo e índice vivem dentro do `SKILL.md`.

```markdown
---
name: my-skill
indexed-skill: tier 1
---
# My Skill

Descrição breve.

<!-- INDEX
@auth-overview | authentication | OAuth2, JWT, API Key setup
@auth-jwt | jwt | Token generation and validation
@billing | billing | Stripe integration and webhooks
-->

<!-- SECTION:auth-overview | keywords: auth,login,oauth | Visão geral de autenticação -->
## Authentication Overview
...conteúdo...
<!-- /SECTION:auth-overview -->

<!-- SECTION:auth-jwt | keywords: jwt,token,refresh | Implementação JWT -->
## JWT Implementation
...conteúdo...
<!-- /SECTION:auth-jwt -->
```

### Fluxo do agente (Tier 1)
1. Lê frontmatter → detecta `indexed-skill: tier 1`
2. Faz grep por `<!-- INDEX` no SKILL.md → lê as entradas `@id | topic | description`
3. Identifica seções relevantes por topic/keywords
4. Faz grep por `<!-- SECTION:id -->` → obtém line number de início
5. Faz grep por `<!-- /SECTION:id -->` → obtém line number de fim
6. Lê apenas o range de linhas necessário

## Tier 2 — Múltiplos arquivos

O `SKILL.md` contém o índice de arquivos. Cada arquivo contém seus próprios anchors de seção.

### SKILL.md
```markdown
---
name: my-large-skill
indexed-skill: tier 2
---
# My Large Skill

Descrição breve.

<!-- INDEX
@sections/auth.md | authentication | OAuth2, JWT, API Key setup and validation
@sections/billing.md | billing | Stripe integration, plans, webhooks
@sections/deploy.md | deployment | Docker and Kubernetes guides
-->
```

### sections/auth.md
```markdown
<!-- SECTION:auth-overview | keywords: auth,login,oauth | Visão geral de autenticação -->
## Authentication Overview
...conteúdo...
<!-- /SECTION:auth-overview -->

<!-- SECTION:auth-jwt | keywords: jwt,token,refresh | Implementação JWT -->
## JWT Implementation
...conteúdo...
<!-- /SECTION:auth-jwt -->
```

### Fluxo do agente (Tier 2)
1. Lê frontmatter → detecta `indexed-skill: tier 2`
2. Faz grep por `<!-- INDEX` no SKILL.md → identifica arquivos relevantes
3. Faz grep por `<!-- SECTION:` no arquivo alvo → lista seções disponíveis com keywords
4. Identifica seções relevantes
5. Faz grep por `<!-- SECTION:id -->` e `<!-- /SECTION:id -->` → obtém line range
6. Lê apenas o range de linhas necessário

## Formato dos marcadores

### INDEX (no SKILL.md)
```
<!-- INDEX
@{target} | {topic} | {description}
-->
```
- **Tier 1**: `target` = section id (ex: `auth-overview`)
- **Tier 2**: `target` = caminho do arquivo (ex: `sections/auth.md`)

### SECTION (nos arquivos de conteúdo)
```
<!-- SECTION:{id} | keywords: {kw1,kw2,...} | {description} -->
...conteúdo...
<!-- /SECTION:{id} -->
```

### Regex de parsing
| Marcador | Regex |
|---|---|
| Index entries | `@(.+?) \| (.+?) \| (.+)` |
| Section open | `<!-- SECTION:(\S+) \| keywords: (.+?) \| (.+?) -->` |
| Section close | `<!-- /SECTION:(\S+) -->` |

## Ecossistema

| Skill | Propósito |
|---|---|
| `indexed-skill/SKILL.md` | Ensina o agente a **consumir** indexed skills |
| `indexed-skill-creator/SKILL.md` | Ensina o agente a **criar** indexed skills |
